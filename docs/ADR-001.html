Decisión de Arquitectura

Contexto

En nuestro proyecto se busca implementar un servicio en el que el cliente pueda no solo tener sus propias funciones accesibles mediante perfiles y permisos, sino también crear clases/cursos y compartir materiales didácticos. Para lograrlo, se decidió integrar WordPress, WooCommerce y Moodle, que son plataformas abiertas y gratuitas.

Decisión

Se adopta una plataforma híbrida abierta combinando WordPress, WooCommerce y Moodle, unidas mediante un plugin como Edwiser Bridge, lo que permite flexibilidad y escalabilidad en la gestión de cursos y usuarios.

Beneficios

  • ✔ Automatización en la venta de cursos.
  • ✔ Gestión de usuarios sin intervención manual.
  • ✔ Fluidez en la experiencia del usuario en un solo flujo.
  • ✔ Certificados automáticos personalizados.
  • ✔ Flexibilidad para agregar contenido educativo, productos digitales y físicos.

Riesgos

  • ⚠ Dependencia del plugin Edwiser Bridge.
  • ⚠ Complejidad técnica al mantener tres sistemas.
  • ⚠ Posibles problemas de rendimiento si el hosting no soporta la carga.

Mitigaciones

  • ✔ Hosting de calidad optimizado para WordPress y Moodle.
  • ✔ Monitoreo constante y respaldos frecuentes.
  • ✔ Plugins actualizados y probados.
  • ✔ Membresía premium de Edwiser Bridge para mayor estabilidad.
  • ✔ Plan de contingencia: acceso manual temporal a cursos en caso de fallos.
